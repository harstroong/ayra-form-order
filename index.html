🔐 Akses Login CS
Buat akun login untuk CS supaya mereka bisa follow-up lead yang belum diproses lewat WA. Setiap akun terhubung ke profil CS dari menu CS & Greeting. CS hanya melihat lead dari produk yang ditugaskan ke profilnya — tanpa akses ke pengaturan apapun.
➕ Tambah Akun CS
Profil dibuat di menu CS & Greeting
Huruf kecil, tanpa spasi. Unik per akun.
Password ditampilkan jelas supaya admin bisa kasih ke CS.
💡 Cara Kerja Login CS
  1. Buat profil CS di menu CS & Greeting (nama + nomor WhatsApp)
  2. Assign profil CS ke produk di menu Produk (tab CS & Greeting)
  3. Buat akun login di sini, pilih profil CS-nya, kasih username & password
  4. CS login lewat halaman login utama → tab CS
  5. CS lihat lead yang masuk untuk produknya, tandai status follow-up & catat hasilnya

📊 Google Tag Manager
Hubungkan GTM untuk tracking konversi Google Ads secara otomatis
GTM Belum Terhubung
Masukkan Container ID untuk mengaktifkan tracking
Temukan di GTM dashboard → Admin → Container Settings
Untuk direct tracking tanpa GTM. Format: AW-ID/label
📋 Events yang Di-track
generate_lead
Form Submitted
Dikirim saat customer mengisi & submit form. Berisi: nama, produk yang dipilih.
page_view
Halaman Dilihat
Otomatis dikirim GTM saat halaman landing page dimuat.
conversion
Google Ads Conversion
Dikirim ke Google Ads saat form disubmit (jika Conversion Label diisi).
🛠️ Cara Setup GTM untuk Google Ads
  1. Buat akun GTM di tagmanager.google.com
  2. Buat Container baru → copy Container ID (GTM-XXXXX) → paste di atas
  3. Di GTM, buat Trigger: Custom Event → Event Name: generate_lead
  4. Buat Tag: Google Ads Conversion Tracking → pilih trigger tadi
  5. Publish container → selesai! Setiap lead otomatis ter-track di Google Ads

☁️ Cloud Sync (Firebase)
Sinkronkan data ke cloud supaya CS bisa login dari HP & admin lihat real-time dari laptop
Belum terhubung
🛠️ Cara Setup Firebase (Sekali Aja, 5 Menit)
  1. Buka console.firebase.google.com → login pakai akun Google
  2. Klik Add project → kasih nama (misal: toko-saya) → Continue → Continue → Create
  3. Di project, klik Build → Firestore Database → Create database
  4. Pilih lokasi terdekat (misal asia-southeast2) → Start in test mode → Enable
  5. Setelah Firestore aktif, kembali ke Project Overview (klik gear icon ⚙️)
  6. Klik Project settings → General → scroll ke bawah → Your apps → klik ikon </> (Web)
  7. Kasih nama app → Register app → akan muncul firebaseConfig
  8. Copy nilai apiKey dan projectId dari firebaseConfig → paste ke form di bawah ⬇️
  9. Klik Test Connection → kalau hijau, klik Save & Enable Cloud Sync
⚠️ Test mode = data bisa dibaca/ditulis oleh siapapun yang punya project ID — OK buat awal/testing. Untuk produksi, nanti Firestore Rules-nya bisa dikencengin (kontak gua kalau perlu bantu).

Tools buat diagnostik dan recovery kalau data kelihatan ilang.
Cek Cloud vs Local — bandingin jumlah record di cloud sama di browser ini.
Force Upload — timpa cloud pakai data lokal (kalau cloud kosong padahal local punya).
Restore Backup — balikin data dari snapshot lokal terakhir (otomatis disimpan sebelum sync overwrite).
